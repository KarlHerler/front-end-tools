About

I have collected some of the front-end development tools that I use on a regular basis. This list is still a work in progress and I would love any contributions to it. At the moment if you want to contribute just head over to github where you will find the entire source for this project.

Frameworks - javascript

This section holds larger tools that have an impact on several parts of your project.

jQuery

jQuery is a must have javascript framework.

CoffeeScript

CoffeeScript is a fairly new addition to my stack.

Backbone.js

Backbone.js is also a fairly new addition to my stack. Backbone provides a Model-View-Controller (or Model-View-Collection as they call it) architecture to your webapp. Backbone also easies your communication to the server and provides you with a solid way of declaring your datastructures.

Backbone isn't a tool that you will need on all your projects but if you are heading into (or are in) a web app project you really want to use backbone.

html5boilerplate

Html5boilerplate is so much more than just a boilerplate, it is all you need to have to get a solid start to any project.

Frameworks - Visual

These

Bootstrap

Twitter recently launched a visual framework called "bootstrap", it is fairly lightweight, at the moment it might even be a bit too lightweight. It is the visualframework that powers most of this thing that you are looking at right now, albiet with some tweaks.

Blueprint

Blueprint is the mother of all css frameworks, it has everything you would need to start working on a site. It is a great tool for you if you aren't all that familiar with css and just want something that looks good out of the box.

jQuery UI

jQuery UI is a complement to jQuery which offers User Interface components for most of your needs. I'll mention some specific components in other sections because they are good enough to stand on their own.

If tou want to spice jQuery UI up a bit I recommend taking a look at the Aristo theme for jQuery UI which is a theme designed for the cappucino framework by sofa.

Layouts & Grids

While I personally do not like grid systems I am often forced to work with them and many people like them so they do deserve a mention here. One should also notice that both bootstrap and blueprint provides a grid for usage so if you're swinging with those then you might aswell use that (unless you need a specific size).

960.gs

960.gs is the classic grid system, spanning 960 pixels and avaliable in 12 or 16 columns flavors. It is still very popular, if not the most poplular grid system.

fluidgrid

In a perfect world, all grids would be fluid.

Fluid 960 grid

So sir wants to be fluid and 960? Well I have the thing for you. Actually I don't know anything about this grid, haven't used it but it exists at least.

Typography

Typography is a bit of a obession of mine so this section might get a bit nerdy.

Alerts and dialogs

The usual alert('somethingrather') dialog has long been a nuicence of front-end and people and does not provide a way of informing the user of something non-critical without interrupting her/his workflow.

smoke.js

Smoke.js a pretty little script package with several types of alerts.
