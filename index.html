About

I have collected some of the front-end development tools that I use on a regular basis. This list is still a work in progress and I would love any contributions to it. At the moment if you want to contribute just head over to github where you will find the entire source for this project.

Frameworks - javascript

This section holds larger tools that have an impact on several parts of your project.

jQuery

jQuery is a must have javascript framework. If you don't use jQuery yet then you should stop reading right now and take a week off to learn it, seriously, I'll still be arround when you come back.

CoffeeScript

CoffeeScript is a fairly new but very much appriciated addition to my stack. CoffeeScript is a language which adds a few small things to javascript that do wonders for the beauty and fun of the language. The only downside with it is that you want to compile CoffeeScript, which requires node.js and knowing what the word "compile" means.

Backbone.js

Backbone.js is also a fairly new addition to my stack. Backbone provides a Model-View-Controller (or Model-View-Collection as they call it) architecture to your webapp. Backbone also easies your communication to the server and provides you with a solid way of declaring your datastructures.

Backbone isn't a tool that you will need on all your projects but if you are heading into (or are in) a web app project you really want to use backbone.

Frameworks - Visual

These frameworks vary in their roles, from just prepering the a canvas for you to work on, to providing build tools and documentation.

html5boilerplate

Html5boilerplate is so much more than just a boilerplate, it is all you need to have to get a solid start to any project.

Bootstrap

Twitter recently launched a visual framework called "bootstrap", it is fairly lightweight, at the moment it might even be a bit too lightweight. It is the visualframework that powers most of this thing that you are looking at right now, albiet with some tweaks.

Blueprint

Blueprint is the mother of all css frameworks, it has everything you would need to start working on a site. It is a great tool for you if you aren't all that familiar with css and just want something that looks good out of the box.

Skeleton

Skeleton is a simple and elegant boilerplate framework. It has very good mobile support and offers elegant typography for the typography novice.

jQuery UI

jQuery UI is a complement to jQuery which offers User Interface components for most of your needs. I'll mention some specific components in other sections because they are good enough to stand on their own.

If tou want to spice jQuery UI up a bit I recommend taking a look at the Aristo theme for jQuery UI which is a theme designed for the cappucino framework by sofa.

Layouts & Grids

While I personally do not like grid systems I am often forced to work with them and many people like them so they do deserve a mention here. One should also notice that both bootstrap and blueprint provides a grid for usage so if you're swinging with those then you might aswell use that (unless you need a specific size).

960.gs

960.gs is the classic grid system, spanning 960 pixels and avaliable in 12 or 16 columns flavors. It is still very popular, if not the most poplular grid system.

fluidgrid

In a perfect world, all grids would be fluid.

Fluid 960 grid

So sir wants to be fluid and 960? Well I have the thing for you. Actually I don't know anything about this grid, haven't used it but it exists at least.

Typography

Typography is a bit of a obession of mine so this section might get a bit nerdy.

Google webfonts

Google has been hard at work on making font embedding as easy as possible. Google webfonts is the culmination of all that work. It is arguably the best font embedding service if you do not need big name typefaces, because google webfonts is limited to free or open source typefaces.

typekit

TypeKit was one of innovators in this field and they still have a very good service. If your typeface needs are more on modern typefaces and especially if you need Adobe&tm; typefaces then this is the service for you.

webfonts by fonts.com

If you need to get big name typefaces on your site then this is the service I would recommend, although it has the worlds slowest admin site.

FontSquirrel

If you are into rolling your own webfonts or other vice don't want to use a second service for serving your fonts. Fontsquirrel also curates a large library of open source fonts which are free for commercial use.

Alerts & Modal overlays

Modal overlay is a umbrella term for things like lightboxes, popovers and any type of element which pops up infront of the content to promt the user to react. They are often used for displaying enlarged images but can be used for most things that need to interrupt the users workflow and be annoying. This section also has some (a) alert tool which although it is a modal overlay it doesn't do anything other than work as a notification or a dialog.

There are also a few modal dialogs avaliable in the bootstrap framework which was mentioned earlier.

smoke.js

Smoke.js a pretty little script package with several types of alerts. It is also framework agnostic and packaged to be self supporting. Smoke uses all kinds of html5 and css3 buzzwordtastic goodies to make it fast and easy to work with.

lightbox 2

A Lightbox is a type of modal overlay designed to overlay images over the current page. You have probably seen a lightbox already, if not then head over to the site to try one out. Lightbox uses the protype framework to help this

fancybox

Fancybox is a newer variant of lightbox, it features more good stuff and is generally better. Fancybox is also designed to work with things other than images so if you are planning on using modal overlays to display other things then this is the one I'd recommend, although I would recommend it over lightbox for images as well.

Drop-in Modals by zurb

Zurb has in one of their experiments made a set of pretty nice modal dialogs. While these modal dialogs aren't packaged in a neat package that you can just drop into your project they are still very much worth a look. You should note that they use css3 quite heavily and might not be suitable for all of your projects.

Charting & Graphs

Doing charts and datavisualizations is another of my hobbies.
Generally speaking there are three ways to go about making graphs:

  1. Scalable Vector Graphics (svg) -tools. My favorite method because svg:s are very flexible, fast and easy to work with, but their biggest flaw is that older browsers might not support svg or they might support it only partially
  2. Flash -tools. Flash tools are the older way of making charts and is still quite popular due too the market penetration of flash and support for older browsers (just as long as they have flash installed). If you plan on supporting anything with an "i" as the first letter of it's name and an apple on the back then this tool is unfortunately out of the window, also flash sucks.
  3. Server-side pre-render. This method uses a little help from a server to create the charts and then sends them to front-end as an image. I wont go into so many of these because they aren't strictly front-end tools but there's a plethora of tools depending on what kind of server environment you use, there's also some hosted tools that do it for you.

Highcharts

Highcharts is a tool that fits into the first category. It is free for non-commercial projects but requires a licencing fee for commercial ones, which is the only tool in this list that does so. I would still recommend it over the free alternatives because it is just so much better. It is the tool that I use on mighty.no.de and mixpanel uses for their charting.

g.raphaël

g.raphaël is the graphing library for a vector graphics tool called Raphaël. It also fits into the first category, being a svg generating tool. It was my first choice of tool since it is open source & free but the it was too impractical for my needs, you should go ahead and try it out before jumping to another tool though.

Open Flash Chart 2

Open Flash Chart is, as hinted by its name, a flash based tool. It is also open source and free, but development on it is rather stagnated. If you want a flash based tool that is rather more...flashy then I would recommend taking a look at FusionCharts which is a commercial flash based charting tool.

Google charts

Google chart tools are a bit of all the things, they offer a variety of charts some svg based, some flash and some server served images. They are still very good and flexible, but require you to do all your rendering via googles servers. Although they are unlikely to go down but rendering them yourself might be faster and you have more control over the process.
