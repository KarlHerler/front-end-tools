About

I have collected some of the front-end development tools that I use on a regular basis. This list is still a work in progress and I would love any contributions to it. At the moment if you want to contribute just head over to github where you will find the entire source for this project.

Frameworks & javascript

a short paragraph about the thing.

jQuery

jQuery is a must have javascript framework.

CoffeeScript

CoffeeScript is a fairly new addition to my stack.

Backbone.js

Backbone.js is also a fairly new addition to my stack. Backbone provides a Model-View-Controller (or Model-View-Collection as they call it) architecture to your webapp. Backbone also easies your communication to the server and provides you with a solid way of declaring your datastructures.

Backbone isn't a tool that you will need on all your projects but if you are heading into (or are in) a web app project you really want to use backbone.

html5boilerplate

Html5boilerplate is so much more than just a boilerplate, it is all you need to have to get a solid start to any project.

Alerts and dialogs

The usual alert('somethingrather') dialog has long been a nuicence of front-end and people and does not provide a way of informing the user of something non-critical without interrupting her/his workflow.

smoke.js

Smoke.js a pretty little script package with several types of alerts.
